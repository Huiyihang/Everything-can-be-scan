# 万物皆可扫｜项目开发文档

## 1. 项目概述

### 1.1 项目名称

万物皆可扫

### 1.2 项目定位

一个移动端优先的 H5 应用。用户上传或拍摄一张日常物品照片，例如缺口杯子、旧鞋子、书包、台灯、键盘等。系统识别主要物品，并以该物品第一人称生成一段 **100 字以内** 的趣味日记，同时自然带出一个相关平台服务推荐入口。

当前版本是可运行、可手机演示的 PoC，已接入阿里云百炼视觉模型，并支持服务落地页和淘宝移动端搜索跳转。

---

## 2. 项目目标

### 2.1 已实现能力

1. 用户上传或拍摄物品图片
2. 前端展示扫描中的过渡体验
3. 后端调用视觉大模型返回结构化识别结果
4. 结果页展示：
   - 物品图片
   - 物品名称
   - 物品状态
   - 第一人称趣味日记
   - 情绪标签
   - 推荐服务卡片
5. 支持本地历史记录
6. 支持历史详情回看
7. 支持独立结果页，刷新后可恢复最近一次扫描结果
8. 支持服务落地页
9. 支持淘宝移动端搜索跳转
10. 支持错误处理和重试
11. 支持 Mock 示例体验
12. 支持上传图片前端压缩
13. 支持结果分享图生成
14. 支持 PWA 基础能力
15. 支持模型请求超时
16. 支持按物品状态优化推荐策略

### 2.2 首版不做

- 用户登录系统
- 数据库持久化
- 实时 WebAR
- 实时视频流识别
- 多物体检测
- 复杂后台管理
- 淘宝客转链、佣金追踪、真实价格库存同步

## 3. 技术方案

### 3.1 技术栈

- Next.js App Router
- React
- TypeScript
- Tailwind CSS
- lucide-react
- localStorage / sessionStorage
- PWA Web App Manifest / Service Worker
- 阿里云百炼 DashScope OpenAI 兼容接口

### 3.2 模型配置

使用阿里云百炼兼容 OpenAI 的接口：

```txt
Base URL: https://dashscope.aliyuncs.com/compatible-mode/v1
Model: qwen3-vl-32b-thinking
```

`.env.local` 示例：

```env
DASHSCOPE_API_KEY=你的百炼 API Key
DASHSCOPE_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
DASHSCOPE_MODEL=qwen3-vl-32b-thinking
DASHSCOPE_TIMEOUT_MS=30000
SCAN_USE_MOCK=false
```

## 4. 系统架构

项目分为五层：

### 4.1 交互层

负责移动端 H5 页面交互：

- 图片上传
- 拍照
- 扫描过渡动画
- 结果展示
- 历史记录展示
- 历史详情回看
- 服务落地页展示
- 当前结果页刷新恢复

### 4.2 接口编排层

负责：

- 接收图片 data URL
- 校验图片大小和格式
- 调用视觉模型
- 解析模型 JSON
- 生成统一 `ScanResult`
- 按类别映射推荐服务
- 返回前端展示 JSON

### 4.3 AI 理解与生成层

负责：

- 识别主要物品
- 判断物品状态
- 生成第一人称趣味日记
- 输出标准类别
- 输出情绪标签

### 4.4 服务映射层

负责：

- 根据物品类别决定推荐服务内容
- 根据物品状态优先推荐清洁、维修、养护或回收方案
- 维护服务标题、说明、推荐理由、服务项
- 维护淘宝搜索关键词和跳转链接
- 与 AI 文案生成解耦

当前推荐优先级：

| 物品状态 | 推荐方向 |
| --- | --- |
| `dirty` | 清洁护理 |
| `damaged` | 维修、替换件、安全换新 |
| `worn` | 养护、修补、易耗件替换 |
| `idle` | 回收、转卖、收纳、旧物改造 |
| `new` / `used` / `unknown` | 按物品类别推荐 |

### 4.5 本地存储层

负责：

- 使用 `localStorage` 保存最近扫描记录
- 支持历史列表
- 支持历史详情按 id 回看
- 支持清空历史记录
- 使用 `sessionStorage` 保存当前标签页最近一次扫描结果
- 支持结果页刷新恢复，避免扫描完成后因页面刷新丢失当前结果

## 5. 页面设计

### 5.1 首页

包含：

- 标题：万物皆可扫
- 上传图片入口
- 拍照扫描入口
- 示例体验入口
- 历史记录入口

### 5.2 扫描中状态

包含：

- 图片预览
- 扫描线动画
- HUD 风格边框
- 提示文案：AI 正在听它说话...

### 5.3 结果页

包含：

- 物品图片
- 识别来源
- 物品名称
- 物品类别
- 物品状态描述
- 第一人称趣味日记气泡
- 情绪标签
- 服务推荐卡片
- “再扫一个”按钮
- 历史记录入口

结果页是独立路由 `/result`。扫描成功后，前端会把本次结果写入 `sessionStorage` 并跳转到该页面；同一标签页内刷新页面仍能恢复当前结果。完整记录仍会同步写入 `localStorage` 历史列表。

### 5.4 服务落地页

- 服务标题
- 模拟平台名称
- 服务说明
- 推荐理由
- 淘宝入口商品卡 3 个
- 主 CTA：跳转淘宝移动端搜索
- 返回首页入口

### 5.5 历史记录页

包含：

- 最近扫描记录列表
- 缩略图
- 物品名称
- 日记摘要
- 扫描时间
- 清空历史记录按钮
- 空状态提示

### 5.6 历史详情页

包含：

- 原始图片
- 物品名称
- 物品状态
- 完整日记
- 情绪标签
- 服务推荐卡片
- 返回历史入口

## 6. 错误处理

当前已处理：

- 非图片文件
- 图片过大
- 上传前图片压缩失败
- API 网络失败
- 模型调用失败
- 模型请求超时
- 模型返回格式异常
- 模型图片尺寸/格式限制
- localStorage 写入失败降级

失败时前端提供：

- 轻量提示
- 重新扫描
- 换张照片
- 查看示例结果

## 7. 图片压缩与分享图

### 7.1 图片压缩

用户上传或拍照后，前端会先对图片进行压缩：

- 最长边控制在约 1280px
- 输出 JPEG
- 降低 base64 请求体积
- 减少模型接口失败概率

若原图超过限制或压缩后仍过大，会提示用户换一张更小、更清晰的照片。

### 7.2 分享结果图

结果页和历史详情页支持生成分享图：

- 使用浏览器 Canvas 本地生成 PNG
- 包含物品图、物品名称、日记、情绪标签、服务推荐
- 支持系统分享能力时调用 Web Share
- 不支持时自动下载 PNG

### 7.3 PWA 支持

项目已加入 PWA 基础配置：

- Web App Manifest
- 应用图标和 maskable 图标
- 移动端主题色
- 轻量 service worker
- 客户端注册逻辑

部署到 HTTPS 环境后，可通过手机浏览器添加到主屏幕。本地局域网 HTTP 访问时，service worker 可能受浏览器安全策略限制。

## 8. 淘宝跳转说明

当前版本使用淘宝移动端搜索链接：

```txt
https://main.m.taobao.com/search/index.html?q=关键词
```

这不是淘宝客转链，也不包含佣金追踪、价格库存同步和优惠券能力。后续如需商业化，需要接入淘宝开放平台或阿里妈妈淘宝客。
